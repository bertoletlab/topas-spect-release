Install & build

OpenTOPAS-SPECT is a set of OpenTOPAS extensions: self-contained C++ classes that OpenTOPAS compiles into its own binary. You install it by pointing an OpenTOPAS build at the source directory. There is nothing to install separately, and the OpenTOPAS core is never modified.

Prerequisites

  • OpenTOPAS 4.2.p2 (the version this package is built and tested against).
  • Geant4 11.2.2 (geant4-11-02-patch-02) with its data files, built as OpenTOPAS requires.
  • CMake ≥ 3.16 and a C++17 compiler.
  • For the Python helper scripts only: Python 3.9+ (pip install -r requirements.txt). The simulation itself needs none of these.

Get the source

git clone https://github.com/bertoletlab/topas-spect-release.git

Build

Point a fresh OpenTOPAS build at the extension directory with TOPAS_EXTENSIONS_DIR. OpenTOPAS compiles every .cc it finds and registers the new components and scorers automatically.

cmake -S /path/to/OpenTOPAS \
      -B /path/to/OpenTOPAS-build \
      -DTOPAS_EXTENSIONS_DIR="/path/to/topas-spect" \
      -DGeant4_DIR=/path/to/geant4/lib/cmake/Geant4 \
      -DCMAKE_BUILD_TYPE=Release

cmake --build /path/to/OpenTOPAS-build --parallel 8
Use a fresh build directory per extension set. Reusing a directory configured for a different TOPAS_EXTENSIONS_DIR can carry stale paths and cause confusing link errors.

Verify

Run a shipped example from its own folder. OpenTOPAS resolves IncludeFile paths relative to the directory you launch from, and each example's includes point to ../../systems, ../../phantoms:

cd topas-spect/examples/system
/path/to/OpenTOPAS-build/topas symbia_lehr_sensitivity.txt

You should see the geometry build with no overlap warnings and a sensitivity in counts per decay. To check every shipped example at once:

TOPAS_BIN=/path/to/OpenTOPAS-build/topas bash tests/smoke_decks.sh

Full details, including combining with other extensions, are in the documentation.
